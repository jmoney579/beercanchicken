Networking


The following excerpt from FileMaker's Knowledge Base will help you to take advantage of EGEMS Networking capabilities. Remember the size of the network is limited by the number of computers with FileMaker software. It is recommended that for institutions not able to equip many computers with FileMaker that they dedicate one computer as a "loaner" and allow users to login to EGEMS through the network on this computer.


A FileMaker Pro file can be setup on a network so that others can access the file at the same time.  This allows for simultaneous access to the file using any of these methods:

  • FileMaker Pro for Microsoft Windows or Mac OS X
  • FileMaker Go for iPhone or iPad
  • A compatible web browser (if Instant Web Publishing and/or Custom Web Publishing is enabled)

There are three ways you can share a FileMaker Pro file.

FileMaker Network:  This type of file sharing in FileMaker Pro is also referred to as peer-to-peer file sharing.  Using this type of file sharing does not require the use of a server and is independent of any other types of file sharing that might be part of your operating system.  FileMaker networking also does not require that you have file-level access to the files.  Up to nine concurrent users (five if using FileMaker Pro 8.5 or earlier) are supported using FileMaker Networking.  This type of file sharing is also supported for those users using FileMaker Go.  FileMaker Go users accessing a file using this method are counted against the number of concurrent users mentioned above.

Web Sharing:  Every copy of FileMaker Pro includes a small web server that allows you to share your FileMaker Pro files on the Internet so that anyone with a compatible web browser can access them.  This technology is called Instant Web Publishing.  Instant Web Publishing requires no knowledge of HTML or other web technologies.  FileMaker Pro automatically converts your layouts and business logic into browser compatible layouts.

FileMaker Server:  When more than nine concurrent users are needed or additional security and performance is a priority; then you will want to consider FileMaker Server.  FileMaker Server also provides for automated backups and supports other web technologies like Custom Web Publishing and XML.  FileMaker Go users can connect to a file that is hosted using FileMaker Server.

Regardless of what method you use to share your FileMaker Pro files, there are some common terms you should be familiar with:

Host:  This term refers to the first user that opens up a file in a FileMaker Network peer-to-peer environment.  Subsequent users who open the same file will automatically become “guests” of the host machine.  In this type of environment, if the host decides to close the file, then all the connected clients will also be forced to close the file.  The host can also be a FileMaker Server machine.  FileMaker Go cannot be used to host a file.

Client:  A user becomes the client of a file when they open a file that is already open by a host.  A user typically becomes the client of a file by:

  • clicking the FILE > OPEN REMOTE menu option in FileMaker Pro
  • tapping OPEN FILE BROWSER > LOCAL NETWORK HOSTS on FileMaker Go for iPhone
  • tapping the LOCAL NETWORK HOSTS button (a database icon with a magnifying glass) on FileMaker Go for iPad

 The action of opening up a file that is shared can also be referred t as becoming a “guest” of a file.

Once a shared file is open, the host and clients have access to the same information (if their access privileges allow for it.)  Modifications to the data made by any user are saved to the shared file.
