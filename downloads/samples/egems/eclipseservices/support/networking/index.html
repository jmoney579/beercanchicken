Data Sharing



With the EGEMS Network Package you have the ability to share your database online through Instant Web Publishing, or inter-institutionally through Peer-to-Peer networking. Select a feature below to learn more!


  • Instant Web Publishing

  • Peer-to-Peer Networking

Regardless of what method you use to share your FileMaker Pro files, there are some common terms you should be familiar with:


Host:  This term refers to the first user that opens up a file in a FileMaker Network peer-to-peer environment.  Subsequent users who open the same file will automatically become “guests” of the host machine.  In this type of environment, if the host decides to close the file, then all the connected clients will also be forced to close the file.  The host can also be a FileMaker Server machine.  FileMaker Go cannot be used to host a file.

Client:  A user becomes the client of a file when they open a file that is already open by a host.  A user typically becomes the client of a file by:

  • clicking the FILE > OPEN REMOTE menu option in FileMaker Pro
  • tapping OPEN FILE BROWSER > LOCAL NETWORK HOSTS on FileMaker Go for iPhone
  • tapping the LOCAL NETWORK HOSTS button (a database icon with a magnifying glass) on FileMaker Go for iPad
The action of opening up a file that is shared can also be referred to as becoming a “guest” of a file.

Once a shared file is open, the host and clients have access to the same information (if their access privileges allow for it.)  Modifications to the data made by any user are saved to the shared file.



Instant Web Publishing

Instant Web Publishing allows you to share information about your materials over the Internet so that anyone with a compatible web browser can view them. It is  recommended that web publishing be created with guest access only. Guest accounts allow a user to view limited amount of information about the materials and they do not allow editing of any kind. 


This feature is especially useful for institutions currently looking for a way to meet the new National Science Foundation (NSF) data management mandate. The most frequently asked question in regards to the NSF mandate is:


What constitutes “data” covered by a Data Management Plan?

According to NSF: What constitutes such data will be determined by the community of interest through the process of peer review and program management. This may include, but is not limited to:
data, publications, samples, physical collections, software and models.

The following excerpt from FileMaker's Knowledge Base will help you to take advantage of EGEMS Instant Web Publishing capabilities.


Instant Web Publishing, also commonly referred to as IWP, is a fast and easy way of publishing your data to the Internet without having to create web pages or worry about configuring a web server by allowing your copy of FileMaker Pro to act as a “mini” web server.  IWP automatically converts all the layouts in your FileMaker Pro file into web pages.  FileMaker Pro supports up to five concurrent IWP users.

In order to share your files using IWP it is recommended that you have the following:

Static IP address:  A static IP address makes it easier for you to share your files using IWP.  Using a dynamic IP address with IWP would require you to communicate with your users every time the IP address on the host machine changed.


A permanent connection to the Internet:  Provides for round the clock access to your data.


The method for sharing a FileMaker Pro file using IWP is much the same as sharing a FileMaker Pro file using FileMaker Networking.

  •  Open the file in FileMaker Pro that you wish to share using Instant Web Publishing.
  •   Choose FILE > SHARING > INSTANT WEB PUBLISHING
  •   On the INSTANT WEB PUBLISHING dialog box, click ON to enable Instant Web Publishing in FileMaker Pro. When Instant Web Publishing is enabled you will see the URL for your system displayed on this dialog box. This is the URL that others will use to access your shared files.


The INSTANT WEB PUBLISHING dialog box is also where you individually configure web access to any the files you currently have open in FileMaker Pro.  


  •   ALL USERS: Provides access to all FileMaker Pro users on your network
  •   SPECIFY USERS BY PRIVILEGE SET: Limits network access to users based on their privilege set
  •   NO USERS:  Prevents any access using Instant Web Publishing. This is the default setting for a new FileMaker Pro file.


If you require additional access control to Instant Web Publishing or want to keep a record of activity and errors, then consider using the Advanced Options of Instant Web Publishing.  You can access the following Advanced Options by clicking SPECIFY on the INSTANT WEB PUBLISHING dialog box.

TCP/IP Port Number:  The default setting for this is 80.  If you do not have any other web services running on your computer then there is little reason to change this from the default setting.  But, if you are running Instant Web Publishing on a system that has other services running and there is a conflict with Port 80, then Instant Web Publishing can be configured to use a different port by changing this value.  FileMaker, Inc. has registered port number 591 with the Internet Assigned Numbers Authority (IANA) specifically for web publishing.

NOTE: If a port other than 80 is used, FileMaker Pro will automatically append the port number to the end of the URL that is displayed in the INSTANT WEB PUBLISHING dialog box.  Anyone connecting to your site / database will not only have to enter in the URL but will also have to append the non-default port number to the end of the URL or server name.  Specifying the port number is not required if the default port number 80 is used.

Accessible only from these IP addresses:  This option allows you to limit the machines / IP addresses that can access your shared FileMaker Pro databases.  Simply add the IP addresses, separated by commas, of the systems that can access your FileMaker Pro shared files.  Wildcards are supported as well so you can type in 192.168.21.* to allow IP addresses that match the first three sets of numbers access to your files.


Logging Options:  Use these options if you want to keep track of what is going on in your database and any errors that might occur.  If you enable this feature, FileMaker Pro will automatically create log files in the WEB LOGS folder.  The “applicatons.log” captures errors in your scripts as well as web publishing errors like missing databases.  The “access.log” file keeps track of everyone who uses your database.


Session Management:  This setting allows you to log someone off if they have not done anything in a while.  A common scenario might be someone logging into your database and then switching to a different website without properly logging out.  Disconnecting inactive users prevents users of your databases from typing up connections when they are not doing anything.




Networking


The following excerpt from FileMaker's Knowledge Base will help you to take advantage of EGEMS Networking capabilities. Remember the size of the network is limited by the number of computers with FileMaker software. It is recommended that for institutions not able to equip many computers with FileMaker that they dedicate one computer as a "loaner" and allow users to login to EGEMS through the network on this computer.


A FileMaker Pro file can be setup on a network so that others can access the file at the same time.  This allows for simultaneous access to the file using FileMaker Pro for Microsoft Windows or Mac OS X:


This type of file sharing in FileMaker Pro is also referred to as peer-to-peer file sharing.  Using this type of file sharing does not require the use of a server and is independent of any other types of file sharing that might be part of your operating system.  FileMaker networking also does not require that you have file-level access to the files.  Up to nine concurrent users (five if using FileMaker Pro 8.5 or earlier) are supported using FileMaker Networking.
