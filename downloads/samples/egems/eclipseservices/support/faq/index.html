How do I import images and have them link to their corresponding records?

There are two ways to import images, singly (one file) or in batches (one folder). Below are explanations for how to import both ways.


Note, either way the record will need to have a catalog number entered first and for bath importing, the images will need to be named after the catalog number. This "number" doesn't have to be numerical but it is what creates the relationship to the images so it must be entered first.   


To import a single image:

Go to the full screen for the material you wish to import an image to, and select the "image" tab. Click "insert image" and select the image you wish to import. Click "open" and the image should appear in the next available space. If all four spaces are filled you will need to delete an image in order to add new ones.


To import multiple images:

Go to the image browse layout of the material you wish to import images of. Click on the "import" button in the navigation bar (blue arrow). This will pop-up a menu where the "specify" button should be highlighted. Click this button to specify which folder of images you wish to import. Once you have these selected click the "continue" button. 


This will pop-up a screen similar to ones you may have seen when importing your original data, or in the tutorial on importing files. Using the arrow symbols match the fields, for example "Image" > "Image 1". Be sure to match the "File Name" to the "Catalog #". If updating existing records be sure to match the "File Name" to the "Catalog #" with an equals sign and under the Import Action select "update existing records in found set"


Once you have everything set up select "import" and on the following pop-up select "import" again (do not check the box to perform auto-updates) to import your images.


When select "Print Preview" it cuts half of the page off, how do I fix this?

Most likely you are printing in a “portrait” orientation. Select “File” and “Print Setup.” A new box will pop up and under “Orientation” select “Landscape.” EGEMS was designed to print all the essential information for a material by using the Quick Screen layout; however, this must be printed in “landscape” orientation.


When I try to enter data into certain fields I receive an error message saying I have entered an invalid value and “You must enter a valid value”.

If you receive this message you are entering information into a field with an authority file. An indication of this is that these fields have small arrow symbols. If you click on the arrow and what you a wish to enter is not available, or this is the first time you are using this authority file, you must scroll to the bottom of the file and select “edit”. This will bring up an editable version of the file, in which you can add your information. Once this has been added you can then select it from the list.


If you have received this error and there is NO drop down list to edit, you are entering information into a field where the values for the authority file are stored in a separate table. Nearby there will be a grey button that will allow you to access the list and add your information. For example the field “Mineral Group Name” has an accompanying button that says “Add New Mineral Group Name”. Once you have clicked on this button you must select “add new record” to add a new line and enter in the new value.


When I click on a button I am taken to a page that says "No Access". Why does this happen and how do I fix it?

You do not have access to view this page, but for your convenience EGEMS has an easy way to get back to the Home Screen! On the top of the screen there is a small field labeled "Layout". This field has a short drop down list where you can select "Home" or "Help" to take you back to layouts you can access. 


I was using EGEMS over a network connection and I was disconnected, why?


This is more than likely a result of the host closing the file. The host is the first user that opens up a file in a FileMaker Network peer-to-peer environment.  Subsequent users who open the same file will automatically become “guests” of the host machine.  In this type of environment, if the host decides to close the file, then all the connected clients will also be forced to close the file.  The host can also be a FileMaker Server machine. If the timing of this seems odd to you we recommend contacting the host as they may not be aware of the problem.

Why is my EGEMS file size so large all of a sudden (after adding images)?


There is no image size restriction for EGEMS to allow each user to determine the size they feel is best for their specimens and for the purpose of the image. For example, a photo of a rock may be able to be 100kb where as a petrographic image of a thin section from that rock may need to be 50mb in order to show all of the pertinent features. We recommend that you do not upload images that are larger than necessary as the EGEMS file can grow quite large quite fast if many large images (1-5mb or more) are added. Though this does not slow down the program, it does make it harder to make backup copies or move this file if needed and also takes up unnecessary space on the computer.



How come when I export an image, it can't be opened by my computer?


This happens when exporting images from EGEMS on a pc because there is no file extension and, unlike a mac the computer, it can't figure out what file type it is to auto add an extension. To remedy this, when importing the image a pop-up window will ask what you want to save the image as. If you add the file extension to the end of the name you give the file it will save as that type of file, for example "image1.jpeg", and thus the computer will be able to read and open the file without a problem.
